Formal Grammar

An alphabet is a non-empty set of symbols. We can put these symbols together to form formulas.

If we have a set of formation rules (called a formal grammar), we can call the set of formulas that conform to these rules well-formed formulas and the set of all these formulas a formal language.

Specifically, a formal grammar is a set of formation rules, which determine whether formulas are well-formed. These formation rules often take the form of production rules, which is a rewrite rule that can be performed recursively. A rewrite rule is a rule that allows one to replace some strings with other strings (e.g. X=Y → (X)+1=(Y)+1, where X and Y are any well-formed formulas).

A terminal symbol is a symbol for which no production rule exists that can change it into something else. A nonterminal symbol is a symbol that can be replaced by production rules. Example:
Let a formal grammar be defined by
  1. x can become xa
  2. x can become a
"a" is a terminal symbol, because no rule exists which would change it into something else. "x" is a nonterminal symbol, because rules exist

Axiom Schemata

An axiom schema generalizes the notion of axiom and is a formula in the language of an axiomatic system in which one or more schematic variables appear. Schematic variables are metalinguistic constructs that stand for any subformula or term of the system, which may or may not be required to satisfy certain conditions (e.g. certain variables may need to be free or not appear in the subformula or term). A subformula is any part of the input formula

The set of terms is defined inductively as

  1. all variables
  2. any expression f(t1,...,tn) of n arguments in which f is a function of valence n and in which every argument ti is a term
Examples:
  • The induction schema that is part of Peano's axioms
  • The axiom schema of replacement that is part of the ZFC axioms

Rules of inference

A rule of inference is a logical form consisting of a function that takes premises, analyzes their syntax, and returns a conclusion. Typically, a rule of inference will preserve the semantic property of truth.

Example:
"ModusPonens(P⇒Q, P)" returns "Q"

Formal Systems

A deductive system consists of
  1. a set of well-formed axioms or axiom schemata
  2. a set of inference rules
A formal system consists of
  1. a formal grammar
  2. a deductive system

Recursion

A formal system is recursive (i.e. effective) if the set of axioms and rules of inference are decidable sets or semidecidable sets, according to context.

A set is decidable (i.e. recursive or computable) if there is an algorithm that terminates after a finite amount of time and correctly decides whether or not a given number belongs to the set.

A set is semidecidable (i.e. recursively enumerable, computably enumerable, provable, or Turing-recognizable) if there is an algorithm that enumerates the members of S (possibly in infinite time). Equivalently, there could be an algorithm that correctly decides whether an element is in the set, but might give no answer (but not the wrong answer) for elements not in the set.

Formal Proofs

A formal proof is a sequence of well-formed formulas such that each well-formed formula in the sequence is either
  1. an axiom
  2. the product of applying an inference rule on a previous formula in the sequence
The last well-formed formula in the sequence is called a theorem. Thus, a theorem is any well-formed formula that can be proven.

If we can prove theorem Q by starting with axiom P, then we say that "P entails Q" or that "Q is a logical consequence of P" (P⊢Q).

Random Tidbits

A formal system is countable if the number of statements is equinumerous to the natural numbers.
